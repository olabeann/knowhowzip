# 노하우집 MVP 구조도·정책·기능명세

- 기준 화면: index.html, creator-admin.html
- 기준일: 2026-07-22
- 목적: 현재 구현된 정적 화면을 실제 서비스에서도 같은 구조와 정책으로 개발하기 위한 구현 인계
- 현재 범위: 정적 HTML·CSS·JavaScript 화면
- 미구현 연동: DB, 실제 인증, PG, 파일 서버, 알리고, Slack
- 상세 정책과 운영 기준은 이 문서 안에서 함께 관리한다.

## 문서 사용 기준

이 문서는 기획 아이디어 모음이 아니라, 현재 구현된 MVP 화면을 실제 개발자가 동일한 정책과 흐름으로 구현하기 위한 기준 문서다.
개발자는 이 문서의 화면 구조, 상태값, 권한 판단, 예외 처리 기준을 기준으로 프론트엔드와 백엔드 구현을 맞춘다.
기획자는 각 화면이 어떤 운영 정책을 전제로 동작하는지 확인하는 용도로 사용한다.

개발 시에는 아래 순서로 읽는 것을 권장한다.

1. 서비스와 화면 흐름은 1~3장을 기준으로 확인한다.
2. 결제, 수강 권한과 플랫폼 환불 정책은 4장과 14장을 함께 확인한다.
3. 수강생 화면 구현은 5장을 기준으로 확인한다.
4. 크리에이터 관리자 화면 구현은 6장과 13장을 함께 확인한다.
5. 실제 서버 연동과 검증 기준은 8~10장을 기준으로 확인한다.

### 용어 사용 원칙

- `강의 콘텐츠`: 영상, 자료와 강의 순서를 관리하는 학습 자산. 관리자 화면과 데이터 모델에서 사용한다.
- `클래스`: 수강생에게 노출되고 판매되는 단위. 관리자와 수강생 화면에서 모두 사용한다.
- `상품`: 이전 구조의 명칭으로, 신규 관리자 문구와 수강생 문구 및 신규 데이터 모델에서는 사용하지 않는다.
- 권장 데이터 엔터티명은 `LectureContent`와 `Class`이며, 클래스는 정렬된 `contentIds`로 강의 콘텐츠를 참조한다.

## 1. 서비스 정의

노하우집은 크리에이터의 강의 판매와 수강 운영을 한곳에 모으는 클래스 플랫폼이다. 주요 유입은 플랫폼 홈 탐색보다 크리에이터의 인스타그램 등 외부 채널에서 크리에이터·클래스 상세 페이지로 바로 진입하는 흐름을 우선한다.

### 수강생 가치

- 여러 크리에이터의 강의를 한 계정에서 결제하고 관리
- 영상, 자료, 공지, 라이브 일정과 링크를 한곳에서 확인
- 구매 클래스 FAQ 검색으로 문의 전에 빠르게 해결
- 결제·수강·환불 CS를 노하우집에서 일관되게 지원

### 크리에이터 가치

- 강의 제작과 진행에 집중
- 결제, 수강 가능 상태, 안내, FAQ, 알림톡 등 반복 운영 업무를 플랫폼에서 관리
- 크리에이터 개인 문의함 대신 노하우집 운영팀이 CS 담당
- 공개 페이지, 강의 콘텐츠와 판매 클래스를 관리자에서 직접 구성

## 4. 핵심 정책

이 장은 논의 과정이 아닌 현재 화면과 기획에서 확정된 운영 기준만을 정의한다.

### 4.1 강의 콘텐츠와 클래스

- `강의 콘텐츠`는 수강생에게 제공할 영상, 자료, 강의 순서와 콘텐츠 설명을 관리하는 재사용 가능한 학습 자산이다.
- `클래스`는 수강생에게 실제로 노출되고 판매되는 단위다.
- 수강생 화면에서는 판매 단위를 기존처럼 `클래스`라고 표시하며 `상품`이라는 명칭을 사용하지 않는다.
- 클래스에는 클래스명과 소개, 대표 이미지, 모집·수강 기간, 가격, 수강 조건, 운영 안내, 라이브 일정, FAQ, 공개 상태를 저장한다.
- 하나의 클래스에는 강의 콘텐츠를 1개 이상 연결한다.
- 하나의 클래스에 여러 강의 콘텐츠를 연결할 수 있고, 같은 강의 콘텐츠를 여러 클래스에서 재사용할 수 있다.
- 클래스에 연결한 강의 콘텐츠의 순서는 수강생 화면에 표시되는 우선순서가 된다.
- 강의 콘텐츠 수정 사항은 해당 콘텐츠를 연결한 모든 클래스에 동일하게 반영한다.
- 플랫폼 데이터와 사용자 화면에서 기수를 별도 항목으로 관리하지 않는다.
- 기수 표기가 필요한 경우 `경매 낙찰 기초반 14기`처럼 클래스명에 직접 포함한다.
- 모집 정원은 현재 운영 항목에 포함하지 않는다.

### 4.1.1 클래스와 수강 가능 상태

- 클래스는 사용자가 결제하는 대상이자 수강 권한을 생성하는 기준이다.
- 클래스 유형은 개별·묶음 등으로 별도 구분하지 않는다.
- 별도의 회원 등급을 두지 않고 결제 이력과 수강 이력으로 접근 가능 여부를 판단한다.
- 클래스마다 수강 조건을 설정한다.
- 수강 조건은 최소 1개를 선택하며 기본값은 `수강 조건 없음`이다.
- `수강 조건 없음`은 선수 클래스 조건과 동시에 선택할 수 없다.
- 선수 클래스는 1개 이상 복수 선택할 수 있으며, 선택된 클래스 중 하나 이상의 이력을 충족하면 신청할 수 있다.
- 사용자가 클래스를 결제하면 해당 클래스의 수강 권한이 생성되고 연결된 강의 콘텐츠가 내 학습에 표시된다.
- 심화 클래스처럼 기초 클래스 이력이 필요한 경우 상세 페이지는 노출하되, 조건 미충족 시 결제 단계에서 안내하고 결제를 막는다.

### 4.2 수강 가능 상태와 결제

- 결제가 완료된 클래스에 한해 해당 클래스와 연결된 강의 콘텐츠를 볼 수 있게 한다.
- 수강 가능 상태는 결제한 클래스의 영상, 자료, 공지, 일정, 참여 링크, FAQ에 적용한다.
- 결제 내역에서 결제일, 클래스, 금액, 결제 상태를 확인할 수 있어야 한다.
- 결제 취소·환불 완료 시 수강 가능 상태의 유지 또는 회수는 환불 정책에 따라 처리한다.
- 실제 서비스에서는 PG 결제 결과를 서버에서 재검증한 후 수강 가능 상태를 생성한다.

### 4.3 CS 운영

- 결제, 환불, 수강 가능 상태, 플랫폼 이용 문의는 노하우집 운영팀이 담당한다.
- 크리에이터는 강의 제작과 진행에 집중하며 별도의 문의함을 운영하지 않는다.
- 클래스 내용과 운영에 관한 반복 질문은 크리에이터가 등록한 클래스별 FAQ로 안내한다.
- 단순 반복 문의를 줄이기 위해 수강생이 FAQ를 먼저 쉽게 찾을 수 있어야 한다.

### 4.4 FAQ 검색

- 내 학습 상단에 FAQ 검색창을 하나만 제공한다.
- 검색 대상은 해당 사용자가 구매한 모든 클래스의 FAQ다.
- 질문과 답변 내용을 모두 검색한다.
- 검색 결과에는 크리에이터명과 클래스명을 함께 표시해 어느 클래스의 답변인지 구분한다.
- 각 클래스 안에는 해당 클래스의 FAQ 목록을 그대로 유지한다.
- 통합 검색은 클래스별 FAQ를 대체하지 않고, 여러 클래스의 FAQ를 먼저 탐색하는 진입점 역할을 한다.

### 4.5 강의 콘텐츠와 영상

- 강의 콘텐츠는 관리자용 콘텐츠 제목과 콘텐츠 설명을 가진다.
- 콘텐츠 제목은 필수이며, 콘텐츠 설명은 선택 항목이다.
- 콘텐츠 제목은 수강생 판매 화면에 노출하지 않고 클래스명과 구분하기 위한 관리자용 값으로 사용한다.
- 강의 콘텐츠에는 영상이 1개 이상 필요하다.
- 영상은 강의 콘텐츠 안에서 강의 단위로 등록하고 순서를 관리한다.
- 각 강의에는 강의 제목과 영상 파일이 필요하며 강의 설명은 선택할 수 있다.
- 지원 형식은 MP4, MOV, WebM이다.
- 영상 파일은 강의당 최대 2GB로 제한한다.
- 영상 업로드 시 재생시간을 확인해 분 단위로 자동 입력한다.
- 실제 서비스에서는 클라이언트와 서버에서 파일 형식과 용량을 모두 검증한다.

### 4.6 제공 자료

- 제공 자료는 강의 콘텐츠의 선택 항목이며 0개 이상 등록할 수 있다.
- 자료는 강의 콘텐츠 안에서 개별 자료 단위로 등록한다.
- 자료를 추가한 경우 자료 제목과 자료 파일을 입력하며 자료 설명은 선택할 수 있다.
- 지원 형식은 PDF, DOC·DOCX, PPT·PPTX, XLS·XLSX, ZIP, JPG·JPEG, PNG다.
- 자료 파일은 개당 최대 50MB로 제한한다.
- 업로드한 파일의 이름과 용량을 표시한다.
- 자료는 해당 강의 콘텐츠가 연결된 클래스를 결제해 수강 가능 상태가 있는 사용자만 확인하고 다운로드할 수 있다.

### 4.7 라이브 일정

- 라이브는 클래스의 선택 기능이다.
- 클래스에는 라이브가 없거나 한 번 또는 여러 번 있을 수 있다.
- 라이브 회차별로 제목, 진행일, 시작 시간, 진행 시간, 참여 링크, 안내 내용을 등록한다.
- 참여 방식은 별도 도구 선택 없이 참여 링크로 안내한다.
- 라이브 일정은 클래스 수강 시작일부터 수강 종료일까지의 기간 안에서만 등록할 수 있다.
- 라이브 진행일은 클래스 수강 종료일을 절대 넘을 수 없다.
- 날짜 제한은 일정 입력 시점과 클래스 최종 저장 시점에 모두 검증한다.
- 수강생은 내 학습에서 구매한 클래스의 라이브 일정과 참여 링크를 확인한다.

### 4.8 알림톡

- 알림톡은 알리고 API와 카카오 승인 템플릿을 기준으로 발송한다.
- 크리에이터는 승인된 템플릿의 전체 문장을 직접 수정할 수 없다.
- 새 템플릿이 승인되기 전에는 기존 승인 문구를 계속 사용한다.
- 승인 완료 후 알리고 API에 승인된 템플릿 코드를 적용한다.
- 새 템플릿은 승인 완료 후 발송 조건을 새로 충족한 수강생부터 자동 적용하며 기존 수강생에게 소급 발송하지 않는다.
- 알림톡 설정과 내부 승인 상태는 크리에이터의 클래스 생성·수정 화면에 노출하지 않는다.

### 4.9 카카오 간편 로그인

- 모든 로그인 진입점은 동일한 카카오 간편 로그인 화면으로 이동한다.
- 카카오 동의 후 이름과 전화번호를 필수 사용자 정보로 저장한다.
- 사용자는 `마이 > 내 정보`에서 이름과 전화번호를 확인할 수 있다.

### 4.10 공개 채널

- 크리에이터는 프로필 이미지, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개를 관리한다.
- 프로필 상단 배경은 색상으로 설정한다.
- 제공 색상 또는 직접 색상 선택을 지원한다.
- 선택한 색상은 관리자 미리보기와 크리에이터 공개 페이지에 반영한다.

### 4.11 공개와 운영 상태

- 클래스의 공개 상태는 `공개`와 `비공개`로 구분한다.
- 새 클래스는 공개 또는 비공개 상태로 저장할 수 있다.
- 공개 클래스의 수정 내용은 저장 후 공개 페이지에 반영한다.
- 모집 가능 여부와 수강 가능 여부는 공개 상태와 별개로 모집 기간 및 수강 기간을 기준으로 판단한다.
- 클래스 삭제는 운영 데이터와 구매 이력에 영향을 줄 수 있으므로 관리자 확인 절차를 거친다.


## 5. 수강생 기능명세

### 5.1 홈

- 여러 크리에이터의 강의를 한곳에서 관리하고 케어받는 가치 전달
- 대표 크리에이터와 클래스 표시
- 수강신청 CTA는 애매모홈 크리에이터 페이지로 이동
- 카카오 간편 로그인 화면

### 5.2 크리에이터 목록

- 이름·분야 검색
- 크리에이터 목록과 상세 이동

### 5.3 크리에이터 상세

- 프로필 이미지, 배경색, 이름, 핸들, 카테고리, 소개
- 클래스·소개 탭
- 페이지 공유
- 상태·제목·평점·가격을 담은 클래스 카드

### 5.4 클래스 상세·결제

- 클래스 제목, 설명, 수강 기간
- 커리큘럼과 자료 안내
- 신청 전 클래스 FAQ
- 플랫폼 공통 환불 규정 탭
- 로그인 확인 후 결제 팝업
- 고정 결제 수단 안내
- 결제 완료 후 내 학습 이동

실서비스 필요: PG 검증, 주문 저장, 결제 실패 처리, 수강 가능 상태 생성, 환불 상태 동기화.

### 5.5 내 학습

- 구매한 모든 클래스 FAQ의 질문·답변 통합 검색
- 검색 결과에 크리에이터명·클래스명 표시
- 검색 패널 접기·펼치기
- 수강 중·수강 종료 구분
- 영상 목록
- 영상별 상태를 `수강 전`, `수강 중`, `수강 완료`로 표시
- 자료 다운로드
- 공지·단톡방·참고 링크
- 여러 라이브 일정과 참여 링크
- 각 클래스의 고유 FAQ 목록

#### 영상별 학습 상태

- `수강 전`: 사용자가 해당 영상의 플레이 버튼을 누른 적이 없고 실제 재생도 시작되지 않은 상태다.
- `수강 중`: 플레이 버튼을 누른 뒤 영상의 실제 재생 시작 이벤트가 발생한 상태다.
- `수강 완료`: 영상 재생 위치가 마지막 구간인 종료 1초 이내에 도달하거나 영상 종료 이벤트가 발생한 상태다.
- 플레이 버튼 클릭만 기록하지 않고 실제 재생이 시작된 시점을 기준으로 `수강 중`으로 변경한다.
- `수강 완료`가 된 영상은 다시 재생하더라도 이전 상태로 되돌리지 않는다.
- 상태는 사용자와 영상 단위로 저장한다.
- 별도의 `이어보기` 상태나 문구는 표시하지 않고 영상별 상태만 표시한다.

### 5.6 마이

- 결제 내역: 결제일, 주문번호, 클래스, 금액, 결제 상태, 환불 상태
- 결제 내역마다 `환불 요청` 버튼을 표시하고, 선택하면 노하우집 카카오톡 채널 상담을 연다.
- 내 정보: 이름, 전화번호, 로그인 방식, 계정 상태

### 5.7 고객센터

- 노하우집 운영팀의 결제·수강·환불 지원 안내
- 기본 FAQ
- 카카오톡 고객센터
- 카테고리형 전체 FAQ 검색은 MVP 제외

## 6. 크리에이터 관리자 기능명세

### 6.1 대시보드

- 이번 달 매출, 전체 수강생, 등록 클래스
  - 매출은 결제 완료 기준으로 집계한다.
  - 수강생은 해당 크리에이터의 강의를 수강 중인 수강생 수를 집계한다.
  - 클래스는 등록된 클래스 기준으로 집계한다.
- 강의 콘텐츠
  - 최근 등록순으로 최대 3개까지 표시한다.
  - `전체 보기` 선택 시 강의 콘텐츠 목록으로 이동한다.
- 최근 수강 신청
  - 결제가 완료된 수강생을 최근 결제 순으로 최대 4명까지 표시한다.
  - `전체 보기` 선택 시 수강생 관리 목록으로 이동한다.
- ‘내 페이지 보기’로 공개 페이지 이동

제외: 답변 대기 문의, 오늘 할 일, 매출 추이, 새 클래스 만들기 단축 버튼.

### 6.2 강의 콘텐츠 목록

- 강의 콘텐츠 목록은 대표 이미지 없이 정보 중심 카드로 표시한다.
- 카드에는 콘텐츠 제목, 설명, 영상 수, 자료 수와 연결 클래스 수를 표시한다.
- 카드 클릭 시 강의 콘텐츠 수정 화면으로 이동한다.
- 우측 메뉴에는 `수정`, `삭제`를 제공한다.

### 6.3 강의 콘텐츠 생성·수정

강의 콘텐츠 편집 화면은 상단 단계 표시를 통해 원하는 영역으로 이동할 수 있게 한다.

#### 1단계 강의 콘텐츠 정보

- 콘텐츠 제목: 필수
- 콘텐츠 설명: 선택
- 콘텐츠 제목은 관리자용이며 수강생 판매 화면에는 클래스명을 표시한다는 안내를 입력창 아래에 노출한다.

#### 2단계 영상·자료

- 영상: 1개 이상 필수
- 영상별 제목, 설명, 영상 파일과 자동 재생시간을 관리한다.
- 영상 순서가 수강생 화면의 강의 순서가 된다.
- 제공 자료: 선택, 0개 이상
- 자료별 제목, 설명과 파일을 관리한다.

### 6.4 클래스 관리

- 클래스는 수강생에게 실제로 노출되고 판매되는 단위다.
- 클래스 카드에는 클래스명, 소개, 판매 가격, 모집·수강 기간, 수강 조건, 연결 강의 콘텐츠와 공개 상태를 표시한다.
- 클래스 카드 선택 시 해당 클래스 수정 화면으로 이동한다.
- 카드 우측 메뉴에는 `수정`, `복제`, `삭제`를 제공한다.
- 클래스 수정 화면 상단의 미리보기 기능은 유지한다.
- 새 클래스 등록과 수정 화면은 긴 폼이므로 상단에 7단계 이동 메뉴를 고정해 원하는 영역으로 바로 이동할 수 있게 한다.

#### 1단계 클래스 정보

- 전체 항목 필수
- 클래스명, 한 줄 소개, 상세 소개, 대표 이미지, 카테고리, 난이도와 키워드 태그를 입력한다.
- 키워드 태그는 검색과 클래스 노출에 사용하며 1개 이상 등록한다.
- 상세 소개는 스마트 에디터로 작성하고 텍스트 서식, 목록, 링크와 이미지 삽입을 지원한다.
- 기수가 필요한 경우 별도 필드가 아니라 클래스명에 직접 포함한다.

#### 2단계 강의 콘텐츠 연결

- 강의 콘텐츠 1개 이상 필수
- 한 클래스에 여러 강의 콘텐츠를 복수 선택할 수 있다.
- 같은 강의 콘텐츠를 여러 클래스에서 재사용할 수 있다.
- 선택한 콘텐츠에 우선순위를 지정하며 해당 순서대로 수강생 화면에 표시한다.

#### 3단계 수강 조건

- 1개 이상 선택 필수
- 기본값은 `수강 조건 없음`이다.
- `수강 조건 없음`은 선수 클래스 선택 영역과 시각적으로 구분한다.
- `수강 조건 없음`과 선수 클래스는 동시에 선택할 수 없다.
- 선수 클래스는 여러 개 선택할 수 있다.

#### 4단계 모집·수강 기간과 가격

- 고정된 단일 결제 방식이므로 결제 방식 선택 필드는 제공하지 않는다.
- 판매 가격: 필수
- 할인 가격: 선택이며 판매 가격보다 낮아야 한다.
- 최종 결제 금액과 할인율을 즉시 미리 보여준다.
- 모집 시작일과 모집 종료일: 필수
- 수강 시작 방식: 필수
- `지정 기간 수강`은 수강 시작일과 수강 종료일을 필수로 입력한다.
- `결제 즉시 수강`은 결제일부터 적용할 수강 가능 일수를 필수로 입력한다.

#### 5단계 운영 안내·라이브 일정

- 선택 항목
- 단톡방·외부 링크, 운영 안내와 여러 라이브 회차를 등록할 수 있다.
- 라이브 회차별 날짜, 시간, 진행 시간, 참여 링크와 안내를 관리한다.
- 라이브 일정은 클래스 수강 기간 범위 안에서 검증한다.

#### 6단계 FAQ

- 선택 항목이며 새 클래스는 FAQ 0개로 시작한다.
- 질문과 답변을 여러 개 추가·삭제할 수 있다.

#### 7단계 환불 정책

- 환불 정책은 플랫폼 공통 정책으로 관리하며 클래스별 입력·수정을 허용하지 않는다.
- 상단에 `본 규정은 공통으로 적용되며, 수정이 불가능합니다.`라고 안내한다.
- 정책 전문은 기본적으로 접고 `환불 규정 자세히 보기`를 선택했을 때 펼쳐지는 읽기 전용 영역으로 표시한다.
- 같은 접기·펼치기 형식의 정책을 모든 클래스 상세의 `환불 규정` 탭에 표시한다.

#### 결제 이력이 있는 클래스

- 클래스명, 소개, 공개 상태, 운영 안내, 라이브 일정과 FAQ처럼 운영상 필요한 항목만 수정할 수 있다.
- 연결 강의 콘텐츠, 수강 조건, 가격과 수강 기간 등 결제 조건을 바꾸려면 새 클래스를 생성한다.

### 6.5 수강생 관리

- 이름, 전화번호, 최근 결제 클래스, 수강 기간과 상태를 목록에 표시한다.
- 이름·이메일·수강 가능 클래스·수강 이력을 검색할 수 있다.
- 클래스와 수강 상태를 필터링할 수 있다.
- 수강생 관리의 상태는 결제 상태가 아니라 `수강 대기`, `수강 중`, `수강 완료`, `기간 만료` 같은 학습 상태로 표시한다.
- 상단 상태 요약은 `전체 상태`, `수강 대기`, `수강 중`, `수강 종료` 네 항목을 같은 너비로 표시한다.
- 크리에이터는 수강생이 볼 수 있는 클래스와 수강 이력을 확인할 수 있다.
- 환불, 보상 지급, 정산에 영향을 주는 변경은 노하우집 운영자 확인이 필요한 흐름으로 둔다.

### 6.6 매출·정산

- 다음 정산 예정 금액과 예정일
- 월 총 결제
- 환불·취소 금액과 건수
- 클래스별 결제 건수·총매출·정산 예정
- 정산 계좌 관리

제외: 플랫폼 수수료 노출, 정산 내역 다운로드.

### 6.7 채널 설정

공개 채널: 프로필 이미지, 배경색, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개, 공개 페이지 미리보기.

정산 정보: 예금주, 은행, 계좌번호, 정산 이메일, 다음 정산 요약. 정산 정보는 정산일 7일 전까지 변경해야 하며, 이후 변경은 주식회사 위이에 직접 요청한다.

## 9. 정적 화면과 실제 개발의 차이

현재 정적 화면은 화면 구조와 주요 상호작용을 보여준다. 실제 서비스에는 다음이 별도로 필요하다.

- 사용자·크리에이터 인증과 권한
- 서버 데이터 저장
- 주문·결제·환불
- 영상·자료 실제 업로드
- 영상 인코딩·스트리밍
- 파일 다운로드 보안
- 알림톡 승인 상태와 알리고 발송
- Slack 연동
- 날짜·가격·파일의 서버 검증
- 개인정보와 결제 정보 보호

## 10. 개발 검증 규칙

### 강의 콘텐츠

- 콘텐츠 제목 필수, 콘텐츠 설명 선택
- 영상 1개 이상 필수
- 자료 선택
- 영상·자료의 파일 형식과 용량 검증
- 연결 클래스에 영향을 주는 수정임을 저장 전에 안내

### 클래스

- 클래스 정보 전체 항목 필수
- 키워드 태그 1개 이상 필수
- 상세 소개 스마트 에디터 내용 필수
- 강의 콘텐츠 1개 이상 필수
- 수강 조건 1개 이상 필수, 기본값은 `수강 조건 없음`
- `수강 조건 없음`과 선수 클래스 동시 선택 금지
- 판매 가격, 모집 시작일·종료일과 수강 시작 방식 필수
- 할인 가격은 선택이며 판매 가격보다 작아야 함
- 지정 기간 수강은 종료일이 시작일보다 빠를 수 없음
- 결제 즉시 수강은 수강 가능 일수가 1일 이상이어야 함
- 라이브 일정은 수강 기간 밖으로 설정할 수 없음
- 공개 전에 필수 정보 확인

### 파일

- 영상: 허용 형식 및 2GB 제한
- 자료: 허용 형식 및 50MB 제한
- 클라이언트·서버 이중 검증
- 실패 상태와 재시도 제공

### 결제

- 서버에서 PG 결제 상태 재검증 후 수강 가능 상태 생성
- 취소·환불 완료 시 정책에 따라 수강 가능 상태 회수 또는 유지
- 주문·결제·환불 상태 이력 보관

### 알림톡

- 승인 완료 템플릿만 자동 발송
- 재승인 중 기존 승인 템플릿 유지
- 중복 발송 방지 키
- 성공·실패와 재시도 이력 보관

## 12. 기획 결정 요약

1. 핵심 유입은 크리에이터 외부 채널에서 상세 페이지로 바로 들어오는 흐름이다.
2. 수강생은 여러 크리에이터의 강의를 한곳에서 관리한다.
3. 크리에이터는 강의에 집중하고 운영·CS는 노하우집이 지원한다.
4. 공식 구조에서는 기수를 사용하지 않는다.
5. FAQ는 구매 클래스 범위에서 통합 검색하고 클래스별 목록도 유지한다.
6. 라이브는 선택 사항이며 여러 회차를 지원한다.
7. 강의 콘텐츠는 영상·자료·강의 순서를 관리하는 재사용 가능한 자산이며 여러 클래스에 연결할 수 있다.
8. 클래스는 수강생에게 노출·판매되는 단위이며 여러 강의 콘텐츠를 우선순위에 따라 연결할 수 있다.
9. 알림톡 설정과 승인·발송은 플랫폼 운영 흐름에서 관리하며 크리에이터의 클래스 화면에는 노출하지 않는다.
10. 환불 정책은 클래스별 입력이 아니라 플랫폼 공통 정책으로 적용한다.
11. MVP는 불필요한 수치·배지·복잡한 설정을 줄이고 실제 운영 입력에 집중한다.

## 13. 크리에이터 관리자 UI 변경 기준

이 섹션은 크리에이터 관리자 화면에서 확정한 UI/운영 기준을 정리한다.
정적 화면 기준의 설명이며, 실제 서버/DB/결제 연동 시에는 아래 역할 구분을 기준으로 API와 데이터 모델을 맞춘다.

### 13.1 강의 콘텐츠와 클래스 관리 역할 구분

관리자 메뉴와 데이터 모델은 아래 두 역할로 분리한다.

#### 강의 콘텐츠

수강생에게 제공되는 학습 자산을 관리한다.

- 관리자용 콘텐츠 제목과 선택 설명
- 영상 1개 이상과 강의 순서
- 선택 제공 자료
- 콘텐츠 수정·삭제
- 연결된 클래스 수 확인

강의 콘텐츠 목록에는 대표 이미지가 없으므로 커버 영역을 표시하지 않는다.
카드에는 제목, 설명, 영상 수, 자료 수와 연결 클래스 수를 표시하고 우측 메뉴에서 수정·삭제한다.

#### 클래스 관리

수강생에게 실제로 노출되고 판매되는 클래스를 관리한다.

- 클래스명과 소개
- 대표 이미지
- 강의 콘텐츠 연결과 표시 우선순위
- 수강 조건
- 모집·수강 기간
- 판매 가격과 선택 할인 가격
- 운영 안내와 라이브 일정
- FAQ
- 공개 상태
- 플랫폼 공통 환불 정책 안내

기존 클래스 관리의 `1. 클래스 정보`에 있던 판매용 정보는 클래스 관리에 둔다.
수강 기간, 가격, 운영 정보와 노출 여부도 클래스 단위에서 관리한다.

강의 콘텐츠와 클래스는 다대다 관계다.
예를 들어 `경매 기초 커리큘럼` 강의 콘텐츠를 `경매 낙찰 기초반 14기`와 이후 기수 클래스에서 함께 재사용할 수 있다.

### 13.2 수강생 관리 기준

수강생 관리 화면은 수강생에 대한 정보를 확인하는 화면이다.
결제 금액보다는 현재 수강 상태와 최근 결제 클래스 중심으로 보여준다.

목록 컬럼은 다음 기준으로 둔다.

- 이름
- 전화번호
- 최근 결제 클래스
- 수강기간
- 상태

상태 예시는 다음과 같다.

- 수강 중
- 수강 대기
- 수강 완료
- 기간 만료

상태별 수강생은 상단 상태 요약 탭으로 확인한다.
목록은 10명 단위 페이징으로 보여준다.

수강생 목록에서 학생을 클릭하면 학생 상세를 확인한다.
학생 상세에는 현재 크리에이터의 클래스 결제·수강 이력만 표시한다.
다른 크리에이터의 결제/수강 이력은 표시하지 않는다.

### 13.3 매출·정산 기준

매출·정산 화면은 월별 조회를 지원한다.

월을 변경하면 다음 값이 함께 변경된다.

- 정산 예정 금액
- 총 결제 금액
- 환불·취소 금액
- 환불·취소 건수
- 클래스별 매출 표

클래스별 매출 표의 행을 클릭하면 팝업이 아니라 상세 화면으로 이동한다.
결제 수강생이 많아질 수 있으므로 팝업보다 페이지 이동이 적합하다.

상세 화면 상단에는 다음 정보를 보여준다.

- 선택 월
- 클래스명
- 결제 건수
- 총 매출
- 정산 예정 금액

아래에는 결제 수강생 전체 목록을 테이블로 보여준다.

테이블 컬럼은 다음 기준으로 둔다.

- 이름
- 전화번호
- 최근 결제 클래스
- 결제일
- 수강기간
- 상태

결제 수강생 행을 클릭하면 기존 학생 상세 모달로 연결한다.

## 14. 강의 콘텐츠·클래스와 수강 권한 로직

이 섹션은 강의 콘텐츠와 판매 클래스를 분리한 현재 구조를 실제 서버, API와 데이터베이스에 동일하게 구현하기 위한 기준이다.
FE는 화면 표시와 상태 분기에서 아래 개념을 사용한다.

### 14.1 핵심 개념

- 강의 콘텐츠는 영상, 자료와 강의 순서를 가진 재사용 가능한 학습 자산이다.
- 클래스는 수강생에게 노출되고 결제되는 판매·운영 단위다.
- 클래스는 강의 콘텐츠를 1개 이상 연결한다.
- 한 클래스에 여러 강의 콘텐츠를 연결할 수 있고 같은 강의 콘텐츠를 여러 클래스에서 재사용할 수 있다.
- 사용자가 클래스를 결제하면 해당 클래스의 수강 권한을 생성한다.
- 수강 권한이 유효한 동안 클래스에 연결된 모든 강의 콘텐츠를 이용할 수 있다.
- 별도 회원 등급은 두지 않고 결제 이력과 수강 이력으로 접근 가능 여부를 판단한다.

### 14.2 강의 콘텐츠 데이터 기준

강의 콘텐츠는 최소한 아래 정보를 가진다.

- 콘텐츠 ID
- 관리자용 콘텐츠 제목: 필수
- 콘텐츠 설명: 선택
- 정렬된 영상 목록: 1개 이상 필수
- 영상별 제목, 선택 설명, 파일 정보와 재생시간
- 정렬된 제공 자료 목록: 선택
- 자료별 제목, 선택 설명과 파일 정보

강의 콘텐츠에는 대표 이미지, 가격, 모집·수강 기간, FAQ, 운영 안내, 라이브 일정이나 공개 판매 상태를 저장하지 않는다.
강의 콘텐츠를 수정하면 연결 관계를 유지한 채 해당 콘텐츠를 사용하는 모든 클래스에 변경 사항을 반영한다.

### 14.3 클래스 데이터 기준

클래스는 최소한 아래 정보를 가진다.

- 클래스 ID
- 클래스명, 한 줄 소개와 상세 소개
- 대표 이미지, 카테고리, 난이도와 키워드 태그 목록
- 스마트 에디터로 작성한 상세 소개 콘텐츠
- 정렬된 강의 콘텐츠 ID 목록 `contentIds`: 1개 이상 필수
- 수강 조건: `수강 조건 없음` 또는 선수 클래스 ID 목록
- 판매 가격과 선택 할인 가격
- 모집 시작일과 모집 종료일
- 수강 시작 방식: `fixed` 또는 `immediate`
- `fixed`인 경우 수강 시작일과 수강 종료일
- `immediate`인 경우 결제일부터 적용할 수강 가능 일수
- 운영 안내와 라이브 일정: 선택
- FAQ 목록: 선택
- 공개 상태
- 플랫폼 공통 환불 정책 참조

`contentIds` 배열 순서는 수강생 화면의 콘텐츠 표시 우선순서로 사용한다.
강의 콘텐츠와 클래스는 다대다 관계이므로 관계 테이블을 사용하는 경우에도 정렬 순서를 함께 저장한다.

결제 이력이 없는 클래스는 전체 정보를 수정할 수 있다.
결제 이력이 있는 클래스는 결제 당시 조건과 수강 권한에 영향을 주지 않는 범위만 수정한다.
연결 강의 콘텐츠, 수강 조건, 가격이나 수강 기간을 바꿔야 하면 기존 클래스를 수정하지 않고 새 클래스를 생성한다.

### 14.4 클래스 구매 가능 여부

클래스 구매 가능 여부는 공개 상태, 모집 기간, 연결 콘텐츠와 수강 조건을 함께 확인한다.

- 클래스가 비공개이거나 현재 시점이 모집 기간 밖이면 결제할 수 없다.
- 연결된 강의 콘텐츠가 1개 미만이면 공개하거나 결제할 수 없다.
- `수강 조건 없음`이면 별도 선수 조건 없이 결제할 수 있다.
- 선수 클래스 조건이 있으면 사용자의 결제 이력 또는 수강 완료 이력이 선택 조건 중 하나 이상을 충족해야 한다.
- 조건을 충족하지 못한 사용자에게도 클래스 상세는 보여줄 수 있으나 결제 단계에서 안내하고 결제를 막는다.

예를 들어 `권리분석 실전반` 클래스가 `경매 낙찰 기초반`을 선수 조건으로 가진다면, 해당 사용자가 기초반을 결제했거나 수강 완료한 경우에만 신청할 수 있다.

### 14.5 결제 완료 후 수강 권한 생성

결제가 완료되면 주문을 기준으로 결제한 클래스의 수강 권한을 생성한다.
수강 권한은 클래스 ID를 참조하고, 학습 화면은 클래스의 정렬된 `contentIds`를 따라 영상과 자료를 불러온다.

- 지정 기간 수강: 클래스의 수강 시작일과 수강 종료일을 권한 기간으로 사용한다.
- 결제 즉시 수강: 결제 시점부터 클래스에 설정된 수강 가능 일수를 적용한다.
- 동일한 클래스에 결제 권한이 여러 개 있으면 유효한 권한이 하나라도 있는 동안 수강 가능 상태로 본다.

내 학습에는 결제 기록 목록이 아니라 현재 수강 권한이 있는 클래스 목록을 보여준다.

### 14.6 환불과 권한 처리

환불과 취소는 주문 상태와 클래스 수강 권한 상태를 함께 바꾼다.

- 결제 실패: 주문 생성 가능, 수강 권한은 생성하지 않는다.
- 결제 완료: 주문을 결제 완료로 저장하고 클래스 수강 권한을 생성한다.
- 환불 요청: 사용자가 마이의 결제 내역에서 요청하면 노하우집 카카오톡 채널 상담으로 연결하고, 플랫폼 공통 환불 정책에 따라 권한을 유지하거나 임시 제한할 수 있다.
- 환불 완료: 플랫폼 공통 정책에 따라 권한을 회수하거나 종료일을 조정한다.
- 부분 환불: 아래 계산식을 기준으로 운영자가 재생·다운로드 로그를 확인해 처리한다.

#### 환불 요청 처리 흐름

1. 사용자가 `마이 > 결제 내역`에서 대상 주문의 `환불 요청`을 선택한다.
2. 노하우집 카카오톡 고객센터로 연결하고 상담에서 사용자와 주문번호를 확인해 요청을 접수한다.
3. 운영자는 클래스의 수강 시작 방식과 기준일, 결제 금액, 전체 콘텐츠 수, 재생 로그와 다운로드 로그를 확인한다.
4. 공통 환불 규정에 따라 전액 환불, 부분 환불 또는 환불 제한 여부와 금액을 결정한다.
5. 접수 후 3일 이내 원 결제 수단으로 환불을 요청한다. 카카오톡 연결만으로 환불이 즉시 실행되지는 않는다.
6. PG 환불 완료를 확인한 뒤 주문 상태와 수강 권한을 함께 갱신하고 처리 이력을 보관한다.

#### 전액 환불이 가능한 기간

전액 환불 가능 기간은 수강 시작 방식에 따라 구분하며, 아래 환불 제한 기준을 함께 적용한다.

- 지정 기간 수강 클래스
  - 결제일부터 수강 시작일까지 언제든지 전액 환불한다.
  - 수강 시작일부터 7일 이내 전액 환불한다.
  - 예: 7월 5일 시작 클래스를 6월 10일에 결제한 경우 7월 12일까지 전액 환불할 수 있다.
- 결제 즉시 수강 클래스
  - 결제일부터 7일 이내 전액 환불한다.

#### 환불이 제한되는 경우

- 이미 재생한 강의는 환불 대상에서 제외한다.
- 강의에 포함된 제공 자료를 다운로드한 경우 자료를 회수할 수 없으므로 해당 강의를 수강한 것으로 본다.
- 아직 재생하지 않고 자료도 다운로드하지 않은 강의는 환불 대상에 포함한다.
- 재생 및 다운로드 여부는 노하우집 시스템에 저장된 재생 로그와 다운로드 로그를 기준으로 확인한다.

#### 7일이 지난 후 중도 해지

- 이미 수강한 강의 콘텐츠에 해당하는 금액을 제외하고 환불한다.
- `환불 금액 = 결제 금액 − (결제 금액 ÷ 전체 강의 콘텐츠 수 × 수강한 강의 콘텐츠 수)`
- 재생하거나 자료를 다운로드한 강의 콘텐츠는 수강한 강의 콘텐츠 수에 포함한다.

#### 환불 처리

- 신청 후 3일 이내에 결제한 수단으로 환불 처리한다.
- 카드사 사정에 따라 실제 입금 또는 승인 취소까지 추가 기간이 소요될 수 있다.
- 문의는 노하우집 고객센터 카카오톡 채널에서 접수한다.
- 이 정책은 플랫폼 공통으로 관리하며 크리에이터가 클래스별로 수정할 수 없다.

### 14.7 화면 반영 기준

- 크리에이터 공개 페이지에는 클래스명, 소개, 대표 이미지, 가격, 모집·수강 기간과 수강 조건을 표시한다.
- 수강생 화면에서는 `강의 콘텐츠`나 `상품`이 아니라 기존처럼 `클래스`라는 명칭을 사용한다.
- 결제 팝업은 클래스를 기준으로 열고 고정된 결제 수단을 안내한다.
- 내 학습에는 구매 기록이 아니라 현재 수강 권한이 있는 클래스 목록을 표시한다.
- 하나의 클래스에 연결된 여러 강의 콘텐츠는 클래스에 저장된 우선순서대로 이어서 표시한다.
- 같은 강의 콘텐츠가 여러 클래스에 연결되어도 수강생은 자신이 권한을 가진 클래스 안에서 해당 콘텐츠를 이용한다.
- 영상 상태는 재생 전 `수강 전`, 실제 재생 시작 후 `수강 중`, 마지막 구간 도달 후 `수강 완료`로 표시한다.
- 수강 기간이 끝난 클래스는 기간 만료 또는 수강 종료 상태로 분류한다.
