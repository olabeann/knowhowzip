## 1. 서비스 정의

노하우집은 크리에이터의 강의 판매와 수강 운영을 한곳에 모으는 클래스 플랫폼이다. 주요 유입은 플랫폼 홈 탐색보다 크리에이터의 인스타그램 등 외부 채널에서 크리에이터·클래스 상세 페이지로 바로 진입하는 흐름을 우선한다.

### 수강생 가치

- 여러 크리에이터의 강의를 한 계정에서 결제하고 관리
- 영상, 자료, 공지, 라이브 일정과 링크를 한곳에서 확인
- 구매 클래스 FAQ 검색으로 문의 전에 빠르게 해결
- 결제·수강·환불 CS를 노하우집에서 일관되게 지원

### 크리에이터 가치

- 강의 제작과 진행에 집중
- 결제, 수강 권한, 안내, FAQ, 알림톡 등 반복 운영 업무를 플랫폼에서 관리
- 크리에이터 개인 문의함 대신 노하우집 운영팀이 CS 담당
- 공개 페이지와 클래스 콘텐츠를 관리자에서 직접 구성

## 4. 핵심 정책

이 장은 논의 과정이 아닌 현재 화면과 기획에서 확정된 운영 기준만을 정의한다.

### 4.1 클래스 단위

- 서비스의 기본 콘텐츠 단위는 ‘클래스’다.
- 실제 결제 단위는 ‘상품’이다.
- 클래스 안에는 여러 개의 영상 강의, 자료, FAQ, 운영 안내, 라이브 일정을 담을 수 있다.
- 플랫폼 데이터와 사용자 화면에서 기수를 별도 항목으로 관리하지 않는다.
- 크리에이터가 기수 표기가 필요한 경우 클래스 제목에 직접 포함한다.
- 기존 클래스와 유사한 다음 판매 상품은 ‘클래스 복제’로 생성한 뒤 일정·가격·공개 정보를 새로 확인한다.
- 클래스에는 모집 기간, 수강 기간, 가격, 판매 상태를 설정한다.
- 모집 정원은 현재 운영 항목에 포함하지 않는다.
- 클래스 카드에는 현재 결제한 인원만 표시한다.

### 4.1.1 상품과 수강생 등급

- 상품은 사용자가 결제하는 대상이다.
- 상품 유형은 개별, 묶음 등으로 나누지 않는다.
- 상품마다 포함 클래스를 1개 이상 자유롭게 선택할 수 있다.
- 상품마다 결제 후 부여할 수강생 등급을 설정한다.
- 사용자가 상품을 구매하면 수강생 등급과 포함 클래스 수강 권한이 자동 부여된다.
- 노하우집 기본 수강생 등급을 사용한다.
- 추천 기본 등급은 루키, 챌린저, 프로, 마스터, 레전드다.
- 크리에이터별 등급명/단계 커스텀은 후속 단계로 둔다.
- 운영자의 수동 등급 변경은 예외 처리 기능으로 후속 단계에서 검토한다.

### 4.2 수강 권한과 결제

- 결제가 완료된 상품에 한해 수강생 등급과 수강 권한을 부여한다.
- 수강 권한은 구매한 클래스의 영상, 자료, 공지, 일정, 참여 링크, FAQ에 적용한다.
- 결제 내역에서 결제일, 상품, 금액, 결제 상태, 환불 상태를 확인할 수 있어야 한다.
- 결제 취소·환불 완료 시 수강 권한의 유지 또는 회수는 환불 정책에 따라 처리한다.
- 실제 서비스에서는 PG 결제 결과를 서버에서 재검증한 후 권한을 부여한다.

### 4.3 CS 운영

- 결제, 환불, 수강 권한, 플랫폼 이용 문의는 노하우집 운영팀이 담당한다.
- 크리에이터는 강의 제작과 진행에 집중하며 별도의 문의함을 운영하지 않는다.
- 클래스 내용과 운영에 관한 반복 질문은 크리에이터가 등록한 클래스별 FAQ로 안내한다.
- 단순 반복 문의를 줄이기 위해 수강생이 FAQ를 먼저 쉽게 찾을 수 있어야 한다.

### 4.4 FAQ 검색

- 내 학습 상단에 FAQ 검색창을 하나만 제공한다.
- 검색 대상은 해당 사용자가 구매한 모든 클래스의 FAQ다.
- 질문과 답변 내용을 모두 검색한다.
- 검색 결과에는 크리에이터명과 클래스명을 함께 표시해 어느 클래스의 답변인지 구분한다.
- 각 클래스 안에는 해당 클래스의 FAQ 목록을 그대로 유지한다.
- 통합 검색은 클래스별 FAQ를 대체하지 않고, 여러 클래스의 FAQ를 먼저 탐색하는 진입점 역할을 한다.

### 4.5 영상 콘텐츠

- 영상은 클래스 안에서 강의 단위로 등록한다.
- 각 강의에는 강의 제목, 강의 설명, 영상 파일이 필요하다.
- 지원 형식은 MP4, MOV, WebM이다.
- 영상 파일은 강의당 최대 2GB로 제한한다.
- 영상 업로드 시 재생시간을 확인해 분 단위로 자동 입력한다.
- 실제 서비스에서는 클라이언트와 서버에서 파일 형식과 용량을 모두 검증한다.

### 4.6 제공 자료

- 자료는 클래스 안에서 개별 자료 단위로 등록한다.
- 각 자료에는 자료 제목, 자료 설명, 자료 파일이 필요하다.
- 지원 형식은 PDF, DOC·DOCX, PPT·PPTX, XLS·XLSX, ZIP, JPG·JPEG, PNG다.
- 자료 파일은 개당 최대 50MB로 제한한다.
- 업로드한 파일의 이름과 용량을 표시한다.
- 자료는 해당 클래스를 결제해 수강 권한이 있는 사용자만 확인하고 다운로드할 수 있다.

### 4.7 라이브 일정

- 라이브는 클래스의 선택 기능이다.
- 클래스에는 라이브가 없거나 한 번 또는 여러 번 있을 수 있다.
- 라이브 회차별로 제목, 진행일, 시작 시간, 진행 시간, 참여 링크, 안내 내용을 등록한다.
- 참여 방식은 별도 도구 선택 없이 참여 링크로 안내한다.
- 라이브 일정은 클래스 수강 시작일부터 수강 종료일까지의 기간 안에서만 등록할 수 있다.
- 라이브 진행일은 클래스 수강 종료일을 절대 넘을 수 없다.
- 날짜 제한은 일정 입력 시점과 클래스 최종 저장 시점에 모두 검증한다.
- 수강생은 내 학습에서 구매한 클래스의 라이브 일정과 참여 링크를 확인한다.

### 4.8 알림톡

- 알림톡 문구는 크리에이터가 관리자 화면에서 작성하거나 수정한다.
- 크리에이터의 수정 내용은 즉시 발송에 반영하지 않으며 카카오 템플릿 승인을 거친다.
- 새 템플릿이 승인되기 전에는 기존 승인 문구를 계속 사용한다.
- 승인 완료 후 알리고 API에 승인된 템플릿 코드를 적용한다.
- 기본 자동 발송 시점은 다음과 같다.
  - 결제 완료 안내: 결제 완료 즉시
  - 강의 시작 안내: 강의 시작 당일 오전 10시
  - 일정 안내: 일정 당일 오전 10시
  - 강의 종료 7일 전 안내: 종료 7일 전 오전 10시
  - 강의 종료 당일 안내: 종료 당일 오전 10시
- 기본 목록에 없는 알림톡은 크리에이터가 이름, 원하는 발송 상황, 상세 설명을 작성해 노하우집 관리자에게 생성 요청한다.
- 알림톡 생성·수정 요청은 관리자 Slack 채널로 알림을 보낸다.

### 4.9 카카오 간편 로그인

- 모든 로그인 진입점은 동일한 카카오 간편 로그인 팝업을 사용한다.
- 카카오 동의 후 이름과 전화번호를 필수 사용자 정보로 저장한다.
- 사용자는 `마이 > 내 정보`에서 이름과 전화번호를 확인할 수 있다.

### 4.10 공개 채널

- 크리에이터는 프로필 이미지, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개를 관리한다.
- 프로필 상단 배경은 색상으로 설정한다.
- 제공 색상 또는 직접 색상 선택을 지원한다.
- 선택한 색상은 관리자 미리보기와 크리에이터 공개 페이지에 반영한다.

### 4.11 공개와 운영 상태

- 새 클래스는 검토 요청 후 공개하거나 비공개 상태로 저장할 수 있다.
- 공개 중인 클래스의 수정 내용은 저장 후 공개 페이지에 반영한다.
- 클래스 상태는 모집 예정, 모집 중, 마감, 비공개, 운영 중, 수강 종료로 관리한다.
- 클래스 삭제는 운영 데이터와 구매 이력에 영향을 줄 수 있으므로 관리자 확인 절차를 거친다.


## 5. 수강생 기능명세

### 5.1 홈

- 여러 크리에이터의 강의를 한곳에서 관리하고 케어받는 가치 전달
- 대표 크리에이터와 클래스 표시
- 수강신청 CTA는 애매모홈 크리에이터 페이지로 이동
- 카카오 간편 로그인 팝업

제외: 통계 수치, 분야 칩, 라이브·다시보기 배지, 기수 표현.

### 5.2 크리에이터 목록

- 이름·분야 검색
- 크리에이터 목록과 상세 이동
- 검색창과 소개 문구 높이 정렬

제외: 인증 체크, 클래스 수, 누적 수강생, 과도한 사회적 증거.

### 5.3 크리에이터 상세

- 프로필 이미지, 배경색, 이름, 핸들, 카테고리, 소개
- 클래스·소개 탭
- 페이지 공유
- 상태·제목·평점·가격을 담은 클래스 카드

제외: 뒤로가기 라벨, 탭 우측 화살표, BEST·심화·현장 칩, 클래스 수·수강생 수, 인증 마크.

### 5.4 클래스 상세·결제

- 클래스 제목, 설명, 가격, 모집 상태
- 커리큘럼과 자료 안내
- 신청 전 클래스 FAQ
- 로그인 확인 후 결제 팝업
- 결제 수단 선택
- 결제 완료 후 구매 권한 부여와 내 학습 이동

실서비스 필요: PG 검증, 주문 저장, 결제 실패 처리, 수강 권한 자동 부여, 환불 상태 동기화.

### 5.5 내 학습

- 구매한 모든 클래스 FAQ의 질문·답변 통합 검색
- 검색 결과에 크리에이터명·클래스명 표시
- 검색 패널 접기·펼치기
- 수강 중·수강 종료 구분
- 이어서 학습과 영상 목록
- 자료 다운로드
- 공지·단톡방·참고 링크
- 여러 라이브 일정과 참여 링크
- 각 클래스의 고유 FAQ 목록

### 5.6 마이

- 결제 내역: 결제일, 상품, 금액, 결제 상태, 환불 상태
- 내 정보: 이름, 전화번호, 로그인 방식, 계정 상태

### 5.7 고객센터

- 노하우집 운영팀의 결제·수강·환불 지원 안내
- 기본 FAQ
- 카카오톡 고객센터
- 카테고리형 전체 FAQ 검색은 MVP 제외

## 6. 크리에이터 관리자 기능명세

### 6.1 대시보드

- 이번 달 매출, 전체 수강생, 운영 중 클래스
- 클래스 운영 현황
- 최근 수강 신청
- ‘내 페이지 보기’로 공개 페이지 이동

제외: 답변 대기 문의, 오늘 할 일, 매출 추이, 새 클래스 만들기 단축 버튼.

### 6.2 클래스 목록

- 카드: 커버, 상태, 제목, 기간, 결제 인원, 가격
- 카드 클릭 시 수정 화면
- 메뉴: 수정, 미리보기, 복제, 비공개, 삭제 요청
- 미리보기는 현재 화면의 팝업으로 제공
- 프로그레스바와 “현재/정원” 표시 없음
- 별도 관리하기 버튼 없음

### 6.3 클래스 생성·수정

#### 1단계 판매 페이지

- 클래스 제목과 부제·소개
- 기수가 필요하면 제목에 직접 포함
- 대표 이미지, 카테고리, 태그
- 판매 페이지 미리보기

#### 2단계 일정·가격

- 모집 상태
- 모집 시작·마감일
- 수강 시작·종료일
- 판매 가격과 정가
- 수강 종료 후 열람 정책

제외: 클래스 유형 선택, 모집 정원.

#### 3단계 콘텐츠

- 기존 클래스 콘텐츠 불러오기
- 영상: 제목, 설명, 파일, 자동 재생시간, 여러 항목 추가·삭제
- 자료: 제목, 설명, 파일, 여러 항목 추가·삭제

제외: 수강생 제공 혜택 체크 목록.

#### 4단계 수강 운영

- 라이브 진행 여부
- 여러 라이브 회차 추가·삭제
- 회차별 날짜·시간·진행 시간·참여 링크·안내
- 카카오톡 단톡방 링크
- 전체 운영 안내
- 라이브 일정의 수강 기간 내 검증

#### 5단계 FAQ·공개

- 클래스별 FAQ
- 취소·환불 안내
- 검토 요청 후 공개 또는 비공개 저장
- 정보 정확성 확인

### 6.4 상품 관리

- 상품 유형 구분 없이 상품을 생성한다.
- 상품 정보: 상품명, 상품 소개, 노출 상태
- 포함 클래스: 상품 결제 시 권한을 부여할 클래스를 1개 이상 선택
- 수강생 등급: 상품 결제 시 자동 부여할 등급 선택
- 가격·이용 기간: 1회 결제, 판매 가격, 판매 기간, 이용 기간 기준
- 환불·공개 정책: 환불 안내, 결제 완료 안내, 공개 상태
- 상품 카드에는 상품명, 설명, 가격, 이용 기간, 포함 클래스를 표시한다.
- 결제 이력이 있는 상품은 상품 정보만 수정할 수 있다.
- 결제 이력이 있는 상품의 포함 클래스, 부여 등급, 가격, 이용 기간, 환불 정책을 바꾸려면 새 상품을 생성한다.

### 6.5 수강생 관리

- 이름, 이메일, 수강생 등급, 구매 클래스, 결제 금액, 신청일, 상태
- 이름·이메일·등급 검색
- 클래스·등급·상태 필터
- 수강생 관리의 상태는 결제 상태가 아니라 `수강 대기`, `수강 중`, `수강 완료`, `기간 만료` 같은 학습 상태로 표시한다.
- 크리에이터는 수강생 등급을 확인할 수 있다.
- 크리에이터의 직접 등급 수정은 제공하지 않는다.
- 환불, 보상 지급, 운영 예외에 따른 등급 변경은 노하우집 운영자에게 요청하는 흐름으로 둔다.

### 6.6 매출·정산

- 다음 정산 예정 금액과 예정일
- 월 총 결제
- 환불·취소 금액과 건수
- 클래스별 결제 건수·총매출·정산 예정
- 정산 계좌 관리

제외: 플랫폼 수수료 노출, 정산 내역 다운로드.

### 6.7 알림톡 관리

| 알림톡 | 자동 발송 시점 |
|---|---|
| 결제 완료 안내 | 결제 완료 즉시 |
| 강의 시작 안내 | 시작 당일 오전 10시 |
| 일정 안내 | 일정 당일 오전 10시 |
| 강의 종료 7일 전 안내 | 종료 7일 전 오전 10시 |
| 강의 종료 당일 안내 | 종료 당일 오전 10시 |

편집 정보: 안내 문구, 치환 변수, 버튼 이름, 관리자 전달 상세 설명, 미리보기, 템플릿 상태·코드.

승인 정책:

- 문구 수정은 즉시 발송에 반영되지 않는다.
- 카카오 템플릿 승인이 필요하다.
- 재승인 중에는 기존 승인 문구를 계속 사용한다.
- 승인 완료 후 알리고 API 템플릿 코드를 교체한다.
- 생성 요청 시 노하우집 관리자에게 전달하고 Slack 알림을 보낸다.
- 원하는 알림톡이 없으면 이름, 발송 상황, 상세 설명을 새 요청으로 보낸다.

### 6.8 채널 설정

공개 채널: 프로필 이미지, 배경색, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개, 공개 페이지 미리보기.

정산 정보: 예금주, 은행, 계좌번호, 정산 이메일, 다음 정산 요약. 정산 정보는 정산일 7일 전까지 변경해야 하며, 이후 변경은 주식회사 위이에 직접 요청한다.

## 7. 상태 정의

### 클래스

모집 예정, 모집 중, 마감, 비공개, 운영 중, 수강 종료.

### 결제

결제 완료, 결제 취소, 환불 요청, 환불 완료, 결제 실패.

### 알림톡

작성 중, 승인 대기, 승인 완료, 반려, 미등록.

### 콘텐츠

파일 미등록, 업로드 중, 업로드 완료, 처리 실패, 공개, 비공개.

| 카카오 채널 | CS | 노하우집 운영팀 문의 |

## 9. 정적 화면과 실제 개발의 차이

현재 정적 화면은 화면 구조와 주요 상호작용을 보여준다. 실제 서비스에는 다음이 별도로 필요하다.

- 사용자·크리에이터 인증과 권한
- 서버 데이터 저장
- 주문·결제·환불
- 영상·자료 실제 업로드
- 영상 인코딩·스트리밍
- 파일 다운로드 보안
- 알림톡 승인 상태와 알리고 발송
- Slack 연동
- 날짜·가격·파일의 서버 검증
- 개인정보와 결제 정보 보호

## 10. 개발 검증 규칙

### 클래스

- 클래스 제목, 수강 시작·종료일, 판매 가격 필수
- 종료일은 시작일보다 빠를 수 없음
- 라이브 일정은 수강 기간 밖으로 설정할 수 없음
- 판매 가격은 0원 이상
- 공개 전에 필수 정보 확인

### 파일

- 영상: 허용 형식 및 2GB 제한
- 자료: 허용 형식 및 50MB 제한
- 클라이언트·서버 이중 검증
- 실패 상태와 재시도 제공

### 결제

- 서버에서 PG 결제 상태 재검증 후 권한 부여
- 취소·환불 완료 시 정책에 따라 권한 회수 또는 유지
- 주문·결제·환불 상태 이력 보관

### 알림톡

- 승인 완료 템플릿만 자동 발송
- 재승인 중 기존 승인 템플릿 유지
- 중복 발송 방지 키
- 성공·실패와 재시도 이력 보관

## 11. MVP 제외·보류

- 공식 기수 관리
- 모집 정원
- 클래스 카드 프로그레스바
- 클래스 유형 선택
- 크리에이터 문의 관리
- 답변 대기 문의
- 대시보드 매출 추이·오늘 할 일
- 인증 마크와 사회적 증거 수치
- 인기 클래스 푸터 링크
- 고객센터 대규모 FAQ 검색
- 플랫폼 수수료 노출
- 정산 내역 다운로드
- 프로필 배경 이미지

## 12. 기획 결정 요약

1. 핵심 유입은 크리에이터 외부 채널에서 상세 페이지로 바로 들어오는 흐름이다.
2. 수강생은 여러 크리에이터의 강의를 한곳에서 관리한다.
3. 크리에이터는 강의에 집중하고 운영·CS는 노하우집이 지원한다.
4. 공식 구조에서는 기수를 사용하지 않는다.
5. FAQ는 구매 클래스 범위에서 통합 검색하고 클래스별 목록도 유지한다.
6. 라이브는 선택 사항이며 여러 회차를 지원한다.
7. 콘텐츠는 강의·자료 단위로 제목, 설명, 파일을 관리한다.
8. 알림톡은 크리에이터가 문구를 작성하고 승인·발송은 관리자 흐름을 거친다.
9. MVP는 불필요한 수치·배지·복잡한 설정을 줄이고 실제 운영 입력에 집중한다.
