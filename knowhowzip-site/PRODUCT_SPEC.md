# 노하우집 MVP 구조도·정책·기능명세

- 기준 화면: index.html, creator-admin.html
- 기준일: 2026-07-01
- 목적: 현재 구현된 정적 화면을 실제 서비스에서도 같은 구조와 정책으로 개발하기 위한 구현 인계
- 현재 범위: 정적 HTML·CSS·JavaScript 화면
- 미구현 연동: DB, 실제 인증, PG, 파일 서버, 알리고, Slack
- 상세 정책과 운영 기준은 이 문서 안에서 함께 관리한다.

## 문서 사용 기준

이 문서는 기획 아이디어 모음이 아니라, 현재 구현된 MVP 화면을 실제 개발자가 동일한 정책과 흐름으로 구현하기 위한 기준 문서다.
개발자는 이 문서의 화면 구조, 상태값, 권한 판단, 예외 처리 기준을 기준으로 프론트엔드와 백엔드 구현을 맞춘다.
기획자는 각 화면이 어떤 운영 정책을 전제로 동작하는지 확인하는 용도로 사용한다.

개발 시에는 아래 순서로 읽는 것을 권장한다.

1. 서비스와 화면 흐름은 1~3장을 기준으로 확인한다.
2. 결제, 수강 권한, 무료 수강, 환불 정책은 4장과 14장을 함께 확인한다.
3. 수강생 화면 구현은 5장을 기준으로 확인한다.
4. 크리에이터 관리자 화면 구현은 6장과 13장을 함께 확인한다.
5. 실제 서버 연동과 검증 기준은 8~10장을 기준으로 확인한다.

## 1. 서비스 정의

노하우집은 크리에이터의 강의 판매와 수강 운영을 한곳에 모으는 클래스 플랫폼이다. 주요 유입은 플랫폼 홈 탐색보다 크리에이터의 인스타그램 등 외부 채널에서 크리에이터·클래스 상세 페이지로 바로 진입하는 흐름을 우선한다.

### 수강생 가치

- 여러 크리에이터의 강의를 한 계정에서 결제하고 관리
- 영상, 자료, 공지, 라이브 일정과 링크를 한곳에서 확인
- 구매 클래스 FAQ 검색으로 문의 전에 빠르게 해결
- 결제·수강·환불 CS를 노하우집에서 일관되게 지원

### 크리에이터 가치

- 강의 제작과 진행에 집중
- 결제, 수강 가능 상태, 안내, FAQ, 알림톡 등 반복 운영 업무를 플랫폼에서 관리
- 크리에이터 개인 문의함 대신 노하우집 운영팀이 CS 담당
- 공개 페이지와 클래스 콘텐츠를 관리자에서 직접 구성

## 4. 핵심 정책

이 장은 논의 과정이 아닌 현재 화면과 기획에서 확정된 운영 기준만을 정의한다.

### 4.1 클래스 단위

- 서비스의 기본 콘텐츠 단위는 ‘클래스’다.
- 실제 결제 단위는 ‘상품’이다.
- 클래스 안에는 여러 개의 영상 강의, 자료, FAQ, 운영 안내, 라이브 일정을 담을 수 있다.
- 플랫폼 데이터와 사용자 화면에서 기수를 별도 항목으로 관리하지 않는다.
- 크리에이터가 기수 표기가 필요한 경우 클래스 제목에 직접 포함한다.
- 기존 클래스와 유사한 다음 클래스는 ‘클래스 복제’로 생성한 뒤 콘텐츠와 수강 기간을 새로 확인한다.
- 클래스에는 수강 기간과 콘텐츠를 설정한다.
- 모집 정원은 현재 운영 항목에 포함하지 않는다.
- 클래스 카드에는 현재 결제한 인원만 표시한다.

### 4.1.1 상품과 수강 가능 상태

- 상품은 사용자가 결제하는 대상이다.
- 상품 유형은 개별, 묶음 등으로 나누지 않는다.
- 상품마다 결제 후 볼 수 있는 클래스를 1개 이상 자유롭게 선택할 수 있다.
- 별도의 회원 등급을 두지 않고 구매 이력과 수강 이력으로 접근 가능 여부를 판단한다.
- 상품마다 수강 조건을 설정할 수 있다.
- 수강 조건은 `수강 조건 없음` 또는 `기존 클래스 중 1개 이상 선택`을 우선 지원한다.
- 사용자가 상품을 구매하면 포함 클래스가 내 학습에 표시된다.
- 심화 과정처럼 기초 과정 이력이 필요한 상품은 상세 페이지는 노출하되, 조건 미충족 시 결제 단계에서 안내한다.
- 크리에이터는 운영 예외에 따라 특정 수강생을 무료 수강으로 처리할 수 있다.

### 4.2 수강 가능 상태와 결제

- 결제가 완료된 상품에 한해 해당 클래스를 볼 수 있게 한다.
- 수강 가능 상태는 구매한 클래스의 영상, 자료, 공지, 일정, 참여 링크, FAQ에 적용한다.
- 결제 내역에서 결제일, 상품, 금액, 결제 상태를 확인할 수 있어야 한다.
- 결제 취소·환불 완료 시 수강 가능 상태의 유지 또는 회수는 환불 정책에 따라 처리한다.
- 실제 서비스에서는 PG 결제 결과를 서버에서 재검증한 후 수강 가능 상태를 생성한다.

### 4.3 CS 운영

- 결제, 환불, 수강 가능 상태, 플랫폼 이용 문의는 노하우집 운영팀이 담당한다.
- 크리에이터는 강의 제작과 진행에 집중하며 별도의 문의함을 운영하지 않는다.
- 클래스 내용과 운영에 관한 반복 질문은 크리에이터가 등록한 클래스별 FAQ로 안내한다.
- 단순 반복 문의를 줄이기 위해 수강생이 FAQ를 먼저 쉽게 찾을 수 있어야 한다.

### 4.4 FAQ 검색

- 내 학습 상단에 FAQ 검색창을 하나만 제공한다.
- 검색 대상은 해당 사용자가 구매한 모든 클래스의 FAQ다.
- 질문과 답변 내용을 모두 검색한다.
- 검색 결과에는 크리에이터명과 클래스명을 함께 표시해 어느 클래스의 답변인지 구분한다.
- 각 클래스 안에는 해당 클래스의 FAQ 목록을 그대로 유지한다.
- 통합 검색은 클래스별 FAQ를 대체하지 않고, 여러 클래스의 FAQ를 먼저 탐색하는 진입점 역할을 한다.

### 4.5 영상 콘텐츠

- 영상은 클래스 안에서 강의 단위로 등록한다.
- 각 강의에는 강의 제목, 강의 설명, 영상 파일이 필요하다.
- 지원 형식은 MP4, MOV, WebM이다.
- 영상 파일은 강의당 최대 2GB로 제한한다.
- 영상 업로드 시 재생시간을 확인해 분 단위로 자동 입력한다.
- 실제 서비스에서는 클라이언트와 서버에서 파일 형식과 용량을 모두 검증한다.

### 4.6 제공 자료

- 자료는 클래스 안에서 개별 자료 단위로 등록한다.
- 각 자료에는 자료 제목, 자료 설명, 자료 파일이 필요하다.
- 지원 형식은 PDF, DOC·DOCX, PPT·PPTX, XLS·XLSX, ZIP, JPG·JPEG, PNG다.
- 자료 파일은 개당 최대 50MB로 제한한다.
- 업로드한 파일의 이름과 용량을 표시한다.
- 자료는 해당 클래스를 결제해 수강 가능 상태가 있는 사용자만 확인하고 다운로드할 수 있다.

### 4.7 라이브 일정

- 라이브는 클래스의 선택 기능이다.
- 클래스에는 라이브가 없거나 한 번 또는 여러 번 있을 수 있다.
- 라이브 회차별로 제목, 진행일, 시작 시간, 진행 시간, 참여 링크, 안내 내용을 등록한다.
- 참여 방식은 별도 도구 선택 없이 참여 링크로 안내한다.
- 라이브 일정은 클래스 수강 시작일부터 수강 종료일까지의 기간 안에서만 등록할 수 있다.
- 라이브 진행일은 클래스 수강 종료일을 절대 넘을 수 없다.
- 날짜 제한은 일정 입력 시점과 클래스 최종 저장 시점에 모두 검증한다.
- 수강생은 내 학습에서 구매한 클래스의 라이브 일정과 참여 링크를 확인한다.

### 4.8 알림톡

- 알림톡은 알리고 API와 카카오 승인 템플릿을 기준으로 발송한다.
- 크리에이터는 승인된 템플릿의 전체 문장을 직접 수정할 수 없다.
- 알림톡 설정은 별도 메뉴가 아니라 상품 생성·수정 화면 안에서 관리한다.
- MVP에서는 상품별 `결제 후 안내톡` 1개만 제공한다.
- 크리에이터는 상품 안에서 결제 후 안내톡 발송 여부를 켜고 끈다.
- ON인 알림톡만 승인 완료 후 자동 발송 대상이 된다.
- 새로 등록한 상품의 알림톡은 기본 OFF 상태로 시작한다.
- 준비 사항, 외부 링크, 첫 모임 일정이 상품마다 다를 수 있으므로 각 상품은 수정 가능한 항목값을 독립적으로 가진다.
- 특정 강사나 특정 외부 서비스명(예: 카페, 경매 사이트, 설문 도구)을 기본 UI에 고정하지 않는다.
- 수강 준비 안내의 수정 가능 항목은 특정 용도명을 고정하지 않고 `안내항목 제목`과 `안내항목 내용` 쌍으로 제공한다.
- MVP 치환 변수는 수강생명, 상품명, 클래스명, 시작일, 일정, 종료일만 제공한다.
- 크리에이터가 수정할 수 있는 값은 링크, 일정, 버튼명, 짧은 안내값처럼 승인 템플릿 안에 들어가는 항목으로 제한한다.
- 알림톡 변수값에는 이모지를 입력할 수 없다는 안내를 노출한다.
- 템플릿 구조는 변수 입력 영역 위에 항상 표시한다.
- 입력값 적용 예시는 기본으로 숨기고, `예시보기` 버튼을 눌렀을 때 표시한다.
- 새 템플릿이 승인되기 전에는 기존 승인 문구를 계속 사용한다.
- 승인 완료 후 알리고 API에 승인된 템플릿 코드를 적용한다.
- 새 템플릿은 승인 완료 후 발송 조건을 새로 충족한 수강생부터 자동 적용하며 기존 수강생에게 소급 발송하지 않는다.
- 크리에이터 화면은 `상품 수정 → 결제 후 안내톡 켜기 → 상세 설정 노출 → 수정 가능한 항목 설정 → 저장`으로 단순화한다.
- 결제 후 안내톡을 끄면 상세 설정 영역은 숨긴다.
- 카카오 승인 상태, 템플릿 코드, 내부 검수 상태는 크리에이터 화면에 노출하지 않는다.
- 기본 자동 발송 시점은 결제 완료 후 5분 이내다.
- 강의 시작 안내, 일정 안내, 종료 안내, 새 알림톡 요청은 MVP에서 제외한다.

### 4.9 카카오 간편 로그인

- 모든 로그인 진입점은 동일한 카카오 간편 로그인 팝업을 사용한다.
- 카카오 동의 후 이름과 전화번호를 필수 사용자 정보로 저장한다.
- 사용자는 `마이 > 내 정보`에서 이름과 전화번호를 확인할 수 있다.

### 4.10 공개 채널

- 크리에이터는 프로필 이미지, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개를 관리한다.
- 프로필 상단 배경은 색상으로 설정한다.
- 제공 색상 또는 직접 색상 선택을 지원한다.
- 선택한 색상은 관리자 미리보기와 크리에이터 공개 페이지에 반영한다.

### 4.11 공개와 운영 상태

- 새 클래스는 검토 요청 후 공개하거나 비공개 상태로 저장할 수 있다.
- 공개 중인 클래스의 수정 내용은 저장 후 공개 페이지에 반영한다.
- 클래스 상태는 모집 예정, 모집 중, 마감, 비공개, 운영 중, 수강 종료로 관리한다.
- 클래스 삭제는 운영 데이터와 구매 이력에 영향을 줄 수 있으므로 관리자 확인 절차를 거친다.

## 6. 크리에이터 관리자 기능명세

### 6.1 대시보드

- 이번 달 매출, 전체 수강생, 운영 중 클래스
- 클래스 운영 현황
- 최근 수강 신청
- ‘내 페이지 보기’로 공개 페이지 이동

제외: 답변 대기 문의, 오늘 할 일, 매출 추이, 새 클래스 만들기 단축 버튼.

### 6.2 클래스 목록

- 카드: 커버, 상태, 제목, 기간, 결제 인원, 가격
- 카드 클릭 시 수정 화면
- 메뉴: 수정, 미리보기, 복제, 비공개, 삭제 요청
- 미리보기는 현재 화면의 팝업으로 제공
- 프로그레스바와 “현재/정원” 표시 없음
- 별도 관리하기 버튼 없음

### 6.3 클래스 생성·수정

#### 1단계 판매 페이지

- 클래스 제목과 부제·소개
- 기수가 필요하면 제목에 직접 포함
- 대표 이미지, 카테고리, 태그
- 판매 페이지 미리보기

#### 2단계 수강 기간

- 수강 시작·종료일

제외: 클래스 유형 선택, 모집 정원, 모집 기간, 판매 가격, 정가, 수강 종료 후 열람 정책.

#### 3단계 콘텐츠

- 기존 클래스 콘텐츠 불러오기
- 영상: 제목, 설명, 파일, 자동 재생시간, 여러 항목 추가·삭제
- 자료: 제목, 설명, 파일, 여러 항목 추가·삭제

제외: 수강생 제공 혜택 체크 목록.

#### 4단계 수강 운영

- 라이브 진행 여부
- 여러 라이브 회차 추가·삭제
- 회차별 날짜·시간·진행 시간·참여 링크·안내
- 카카오톡 단톡방 링크
- 전체 운영 안내
- 라이브 일정의 수강 기간 내 검증

#### 5단계 FAQ·공개

- 클래스별 FAQ
- 취소·환불 안내
- 검토 요청 후 공개 또는 비공개 저장
- 정보 정확성 확인

### 6.4 상품 관리

- 상품 유형 구분 없이 상품을 생성한다.
- 상품 정보: 상품명, 상품 소개, 노출 상태
- 결제 후 볼 수 있는 클래스: 상품 결제 후 내 학습에 표시할 클래스를 1개 이상 선택
- 수강 조건: 기존 클래스 중 선택해 결제 가능 여부 설정
- 가격·이용 기간: 1회 결제, 판매 가격, 판매 기간, 이용 기간 기준
- 결제 후 안내톡: 발송 여부, 승인 템플릿 변수값, 버튼 이름·연결
- 환불·공개 정책: 환불 안내, 결제 완료 안내, 공개 상태
- 상품 카드에는 상품명, 설명, 가격, 이용 기간, 결제 후 볼 수 있는 클래스를 표시한다.
- 결제 이력이 있는 상품은 상품 정보만 수정할 수 있다.
- 결제 이력이 있는 상품의 결제 후 볼 수 있는 클래스, 수강 조건, 가격, 이용 기간, 환불 정책을 바꾸려면 새 상품을 생성한다.

### 6.5 수강생 관리

- 이름, 이메일, 수강 가능 클래스, 수강 이력, 구매 클래스, 결제 금액, 신청일, 상태
- 이름·이메일·수강 가능 클래스·수강 이력 검색
- 클래스·수강 상태·상태 필터
- 수강생 관리의 상태는 결제 상태가 아니라 `수강 대기`, `수강 중`, `수강 완료`, `기간 만료` 같은 학습 상태로 표시한다.
- 크리에이터는 수강생이 볼 수 있는 클래스와 수강 이력을 확인할 수 있다.
- 크리에이터는 예외적으로 특정 수강생을 무료 수강으로 처리할 수 있다.
- 환불, 보상 지급, 정산에 영향을 주는 변경은 노하우집 운영자 확인이 필요한 흐름으로 둔다.

### 6.6 매출·정산

- 다음 정산 예정 금액과 예정일
- 월 총 결제
- 환불·취소 금액과 건수
- 클래스별 결제 건수·총매출·정산 예정
- 정산 계좌 관리

제외: 플랫폼 수수료 노출, 정산 내역 다운로드.

### 6.7 결제 후 안내톡

| 알림톡 | 자동 발송 시점 |
|---|---|
| 수강 준비 안내 | 결제 완료 후 5분 이내 |

상품 관리 화면 안에서만 제공한다.

MVP 편집 정보: 결제 후 안내톡 발송 여부, 템플릿 구조, 수정 가능한 변수값, 버튼 이름·연결, 예시보기 버튼, 입력값 적용 예시.

상품별 발송 원칙:

- 결제 후 안내톡은 상품 단위로 켜고 끈다.
- ON인 상품을 구매한 수강생에게만 자동 발송된다.
- 새 상품은 OFF 상태로 시작한다.
- 상품별 안내 항목, 링크, 일정, 짧은 안내값이 다를 수 있으므로 각 상품의 알림톡 설정값은 독립적으로 저장한다.
- 발송 시점은 `결제 완료 후 5분 이내`를 사용하고 MVP 화면에서는 세부 설정을 숨긴다.

승인 정책:

- 고정 문구 수정은 크리에이터 화면에서 허용하지 않는다.
- 카카오 템플릿 승인이 필요하다.
- 재승인 중에는 기존 승인 문구를 계속 사용한다.
- 승인 완료 후 알리고 API 템플릿 코드를 교체한다.
- 승인 완료 후 새로 조건을 충족한 수강생부터 자동 발송하며 소급 발송하지 않는다.
- 승인 정책은 운영팀 내부에서 처리하며 크리에이터 화면에는 자세히 노출하지 않는다.
- 새 알림톡 생성 요청은 MVP에서 제공하지 않는다.

### 6.8 채널 설정

공개 채널: 프로필 이미지, 배경색, 이름, 핸들, 카테고리, 한 줄 소개, 상세 소개, 공개 페이지 미리보기.

정산 정보: 예금주, 은행, 계좌번호, 정산 이메일, 다음 정산 요약. 정산 정보는 정산일 7일 전까지 변경해야 하며, 이후 변경은 주식회사 위이에 직접 요청한다.

- 영상·자료 실제 업로드
- 영상 인코딩·스트리밍
- 파일 다운로드 보안
- 알림톡 승인 상태와 알리고 발송
- Slack 연동
- 날짜·가격·파일의 서버 검증
- 개인정보와 결제 정보 보호

## 10. 개발 검증 규칙

- 종료일은 시작일보다 빠를 수 없음
- 라이브 일정은 수강 기간 밖으로 설정할 수 없음

- 영상: 허용 형식 및 2GB 제한

- 실패 상태와 재시도 제공

### 결제

- 서버에서 PG 결제 상태 재검증 후 수강 가능 상태 생성
- 취소·환불 완료 시 정책에 따라 수강 가능 상태 회수 또는 유지
- 주문·결제·환불 상태 이력 보관

### 알림톡

- 승인 완료 템플릿만 자동 발송
- 재승인 중 기존 승인 템플릿 유지
- 중복 발송 방지 키
- 성공·실패와 재시도 이력 보관

## 12. 기획 결정 요약

1. 핵심 유입은 크리에이터 외부 채널에서 상세 페이지로 바로 들어오는 흐름이다.
2. 수강생은 여러 크리에이터의 강의를 한곳에서 관리한다.
3. 크리에이터는 강의에 집중하고 운영·CS는 노하우집이 지원한다.
4. 공식 구조에서는 기수를 사용하지 않는다.
5. FAQ는 구매 클래스 범위에서 통합 검색하고 클래스별 목록도 유지한다.
6. 라이브는 선택 사항이며 여러 회차를 지원한다.
7. 콘텐츠는 강의·자료 단위로 제목, 설명, 파일을 관리한다.
8. 알림톡은 크리에이터가 문구를 작성하고 승인·발송은 관리자 흐름을 거친다.
9. MVP는 불필요한 수치·배지·복잡한 설정을 줄이고 실제 운영 입력에 집중한다.

## 13. 크리에이터 관리자 UI 변경 기준

이 섹션은 크리에이터 관리자 화면에서 확정한 UI/운영 기준을 정리한다.
정적 화면 기준의 설명이며, 실제 서버/DB/결제 연동 시에는 아래 역할 구분을 기준으로 API와 데이터 모델을 맞춘다.

### 13.1 클래스 관리와 상품 관리 역할 분리

클래스 관리는 콘텐츠 관리 중심으로 둔다.

- 클래스 정보
- 영상 커리큘럼
- 제공 자료
- FAQ
- 미리보기
- 복제

MVP 기준으로 클래스 관리 화면에서는 아래 항목을 제거한다.

- 수강 기간 설정
- 공개/비공개 설정
- 수강 운영 설정
- 라이브 일정
- 단톡방 링크
- 운영 안내문
- 마지막 수정일 표시
- 콘텐츠 준비 상태 칩
- 콘텐츠 개수 요약 표시

클래스 복제는 별도 복제 편집 화면으로 이동하지 않는다.
카드 메뉴에서 복제를 누르면 원본 카드 바로 옆에 `클래스명 복제본` 카드가 추가되는 흐름으로 처리한다.

상품 관리는 결제와 수강 권한 판단의 기준으로 둔다.

- 상품명과 소개
- 노출 상태
- 결제 후 볼 수 있는 클래스
- 수강 조건
- 판매 가격
- 판매 시작일/종료일
- 수강 시작일/종료일
- 환불 안내
- 결제 완료 안내

수강 기간과 노출 여부는 상품 단위에서 관리한다.
따라서 클래스는 콘텐츠 묶음이고, 상품은 사용자가 결제하는 대상이자 수강 권한을 만드는 기준이다.

### 13.2 수강생 관리 기준

수강생 관리 화면은 수강생에 대한 정보를 확인하는 화면이다.
결제 금액보다는 현재 수강 상태와 최근 결제 상품 중심으로 보여준다.

목록 컬럼은 다음 기준으로 둔다.

- 이름
- 전화번호
- 최근 결제 상품
- 수강기간
- 상태

상태 예시는 다음과 같다.

- 수강 중
- 수강 대기
- 수강 완료
- 기간 만료
- 무료 수강중

상태별 수강생은 상단 상태 요약 탭으로 확인한다.
목록은 10명 단위 페이징으로 보여준다.

무료 수강 처리는 결제 없이 특정 수강생에게 수강 권한을 부여하는 예외 운영 기능이다.

무료 수강 처리 시 반영 기준은 다음과 같다.

- 최근 결제 상품을 `클래스명 무료 수강` 형태로 표시
- 결제 금액은 0원으로 저장
- 수강 기간은 입력한 시작일/종료일 기준으로 저장
- 상태는 `무료 수강중`으로 표시
- 학생 상세의 결제/수강 이력에도 무료 수강 기록을 추가

무료 수강 처리는 정산이나 환불 금액에 직접 영향을 줄 수 있으므로, 실제 서비스에서는 관리자 권한과 이력 로그가 필요하다.

수강생 목록에서 학생을 클릭하면 학생 상세를 확인한다.
학생 상세에는 현재 크리에이터의 상품/클래스 이력만 표시한다.
다른 크리에이터의 결제/수강 이력은 표시하지 않는다.

### 13.3 매출·정산 기준

매출·정산 화면은 월별 조회를 지원한다.
월 선택 드롭다운은 화면 상단 우측, `정산 계좌 관리` 버튼 왼쪽에 배치한다.

월을 변경하면 다음 값이 함께 변경된다.

- 정산 예정 금액
- 총 결제 금액
- 환불·취소 금액
- 환불·취소 건수
- 클래스별 매출 표

정산 카드 안에는 별도 `정산 계좌 관리` 버튼을 두지 않는다.
계좌 관리 액션은 상단 우측 액션 영역으로 통일한다.

클래스별 매출 표의 행을 클릭하면 팝업이 아니라 상세 화면으로 이동한다.
결제 수강생이 많아질 수 있으므로 팝업보다 페이지 이동이 적합하다.

상세 화면 상단에는 다음 정보를 보여준다.

- 선택 월
- 클래스명
- 결제 건수
- 총 매출
- 정산 예정 금액

아래에는 결제 수강생 전체 목록을 테이블로 보여준다.

테이블 컬럼은 다음 기준으로 둔다.

- 이름
- 전화번호
- 최근 결제 상품
- 결제일
- 수강기간
- 상태

결제 수강생 행을 클릭하면 기존 학생 상세 모달로 연결한다.

### 13.4 FAQ 입력 UI 기준

클래스 FAQ 입력 영역의 질문/답변 입력칸은 플레이스홀더 기준선을 맞춘다.

- 질문 입력칸도 textarea 구조로 사용한다.
- 질문과 답변 플레이스홀더는 모두 상단 정렬한다.
- 질문 입력칸은 크기 조절 없이 고정한다.
- 답변 입력칸은 필요 시 세로 크기 조절을 허용한다.

### 13.5 Git 브랜치와 페이지 공유 기준

`master`와 `main`은 기능 차이가 없다.
둘 다 기본 브랜치로 사용할 수 있다.

페이지 공유 가능 여부는 브랜치 이름 자체가 아니라 배포 설정이 어떤 브랜치와 폴더를 바라보는지에 따라 결정된다.
예를 들어 GitHub Pages라면 `Settings > Pages > Branch`에서 `main`을 선택하면 main 브랜치 기준으로도 페이지 공유가 가능하다.

팀 또는 외부 협업 기준이 `main`이라면 다음 방향이 권장된다.

- 기본 브랜치를 main으로 통일
- Pages 배포 브랜치도 main으로 변경
- 로컬 작업 브랜치와 원격 추적 브랜치도 main 기준으로 정리

`master`를 계속 쓰는 것이 기능상 문제를 만들지는 않지만, 최근 프로젝트에서는 `main`을 기본 브랜치로 쓰는 경우가 많으므로 협업 컨벤션에 맞추는 편이 혼란을 줄인다.

## 14. 상품과 수강 권한 로직

이 섹션은 기존 별도 문서로 관리하던 상품 구조와 수강 권한 로직을 PRODUCT SPEC 안으로 통합한 내용이다.
FE는 화면 표시와 상태 분기에서 아래 기준을 사용하고, 실제 서버 연동 시에는 동일한 개념으로 API 응답을 맞춘다.

### 14.1 핵심 개념

- 클래스는 영상, 자료, FAQ 등 학습 콘텐츠 묶음이다.
- 상품은 사용자가 결제하는 대상이며, 하나 이상의 클래스를 포함할 수 있다.
- 사용자가 상품을 구매하면 상품에 포함된 클래스별 수강 권한이 생성된다.
- 상품 유형은 별도로 나누지 않고, 포함 클래스 수와 수강 조건으로 단일 클래스 상품, 묶음 상품, 단계형 상품을 표현한다.
- 별도 회원 등급은 두지 않고 구매 이력, 수강 이력, 무료 수강 이력으로 접근 가능 여부를 판단한다.

### 14.2 상품 데이터 기준

상품은 최소한 아래 정보를 가진다.

- 상품명과 상품 소개
- 판매 가격
- 노출 상태
- 판매 시작일과 판매 종료일
- 수강 시작일과 수강 종료일
- 결제 후 볼 수 있는 클래스 목록
- 수강 조건
- 환불 안내
- 결제 완료 안내

결제 이력이 없는 상품은 위 정보를 수정할 수 있다.
결제 이력이 있는 상품은 상품명, 상품 소개, 노출 상태만 수정할 수 있다.
결제 후 볼 수 있는 클래스, 수강 조건, 가격, 수강 기간, 환불 정책을 바꾸려면 기존 상품을 수정하지 않고 새 상품을 만든다.

### 14.3 구매 가능 여부

상품 구매 가능 여부는 상품 상태와 수강 조건을 함께 확인한다.

- 상품이 판매 중이 아니면 결제할 수 없다.
- 판매 시작 전이거나 판매 종료 후이면 결제할 수 없다.
- 결제 후 볼 수 있는 클래스가 없으면 결제할 수 없다.
- 수강 조건이 없으면 바로 결제할 수 있다.
- 수강 조건이 있으면 사용자의 구매 이력 또는 수강 완료 이력이 조건을 충족해야 결제할 수 있다.
- 조건을 충족하지 못한 사용자에게도 상품 상세는 보여줄 수 있으나, 결제 단계에서는 안내 문구를 보여주고 결제를 막는다.

예를 들어 권리분석 실전반 상품이 경매 낙찰 기초반 수강 이력을 조건으로 가진다면, 해당 사용자가 경매 낙찰 기초반을 구매했거나 수강 완료한 경우에만 결제할 수 있다.

### 14.4 결제 완료 후 수강 권한 생성

결제가 완료되면 주문을 기준으로 상품에 포함된 클래스별 수강 권한을 생성한다.
FE는 내 학습 화면에서 상품 목록이 아니라 사용자가 볼 수 있는 클래스 목록을 보여준다.

권한 기간은 상품의 수강 시작일과 수강 종료일을 기준으로 표시한다.
동일한 클래스에 대해 여러 상품 또는 무료 수강 권한이 있으면, 유효한 권한이 하나라도 있는 동안 해당 클래스는 수강 가능 상태로 본다.

### 14.5 무료 수강 처리

무료 수강 처리는 결제 없이 운영자가 특정 수강생에게 수강 권한을 부여하는 예외 기능이다.
무료 수강 처리 시 화면과 데이터는 아래처럼 다룬다.

- 최근 결제 상품은 `클래스명 무료 수강` 형태로 표시한다.
- 결제 금액은 0원으로 저장한다.
- 수강 기간은 입력한 수강 시작일과 수강 종료일을 기준으로 저장한다.
- 상태는 `무료 수강중`으로 표시한다.
- 학생 상세의 결제/수강 이력에도 무료 수강 기록을 추가한다.

무료 수강은 정산이나 환불 금액에 직접 영향을 줄 수 있으므로 실제 서비스에서는 관리자 권한과 이력 로그가 필요하다.

### 14.6 환불과 권한 처리

환불과 취소는 주문 상태와 수강 권한 상태를 함께 바꾼다.

- 결제 실패: 주문 생성 가능, 수강 권한은 생성하지 않는다.
- 결제 완료: 주문을 결제 완료로 저장하고 수강 권한을 생성한다.
- 환불 요청: 환불 정책에 따라 권한을 유지하거나 임시 제한할 수 있다.
- 환불 완료: 정책에 따라 수강 권한을 회수하거나 종료일을 조정한다.
- 부분 환불: MVP에서는 운영자 확인 상태로 두고, 상품 단위 환불 정책은 추후 확장한다.

### 14.7 화면 반영 기준

크리에이터 공개 페이지의 상품 카드에는 상품명, 설명, 가격, 수강 기간, 수강 조건, 결제 후 볼 수 있는 클래스를 표시한다.

결제 팝업은 상품을 기준으로 열리며, 결제 완료 안내 문구에는 결제 후 내 학습에서 바로 볼 수 있다는 메시지를 표시한다.

내 학습은 구매한 상품 목록이 아니라 현재 수강 권한이 있는 클래스 목록을 보여준다.
같은 클래스를 여러 상품으로 구매해도 클래스 카드는 하나만 보여준다.
수강 기간이 끝난 클래스는 기간 만료 상태로 분류한다.
